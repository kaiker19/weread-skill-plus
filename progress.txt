## 关键架构决策
- 脚本里没有 LLM。执行模型：脚本拉数据+输出 stdout → agent 读取 → agent 的 LLM 用 prompt 合成 → 推送。
- prompts/ 下的文件是给 agent LLM 的系统指令，不是脚本内部调用。
- 跨书呼应预筛选用 jieba 分词 + SQL LIKE（保持简单），不引入 embedding。
- concepts 表暂不使用，作为 Phase 5 优化保留（需 embedding 或 agent 回写）。
- DB 位置：放 skill 下 data/knowledge.db（env WEREAD_KB_PATH 可覆盖）。未来 web 端应迁移到后端 DB。
